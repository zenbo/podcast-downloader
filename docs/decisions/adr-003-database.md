# ADR-003: データベースの選定

| 項目       | 値         |
| ---------- | ---------- |
| 日付       | 2026-02-21 |
| ステータス | 採用       |

## コンテキスト

番組情報、エピソード情報を永続化するためのデータストアを選定する。

## 検討した選択肢

1. **SQLite** — 軽量な組み込みデータベース。ファイル 1 つで完結。構造化データの検索・管理が得意
2. **JSON ファイル** — シンプル。人が直接読み書き可能。大量データには不向き

## 決定

**SQLite** を採用する。Rust からは **rusqlite** クレートを使用する。

## 理由

- ファイル 1 つで完結し、サーバー不要
- 構造化データの検索・結合が SQL で容易に行える
- ダウンロード履歴の管理など、データ量が増えた場合にも対応可能
- Rust の rusqlite クレートが成熟しており、安定して利用できる
- マイグレーション管理に rusqlite_migration を使用できる

## 影響・リスク

- JSON ファイルに比べて、人がデータを直接確認・編集するのはやや手間がかかる
- ADR-010 により、設定情報は SQLite ではなく JSON（tauri-plugin-store）に保存する方針に変更された

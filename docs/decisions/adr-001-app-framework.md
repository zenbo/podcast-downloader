# ADR-001: アプリケーションフレームワークの選定

| 項目 | 値 |
|------|-----|
| 日付 | 2026-02-21 |
| ステータス | 採用 |

## コンテキスト

Windows 11 向けの GUI デスクトップアプリを開発する。開発環境は macOS であり、将来的にクロスプラットフォームへの拡張を容易にしたい。利用者は開発者本人のみ。

## 検討した選択肢

1. **Electron + TypeScript** — Web 技術で GUI 構築。エコシステム豊富だがバイナリサイズが大きい（100MB 超）
2. **Tauri v2 + Rust** — OS ネイティブ WebView を利用。軽量バイナリ（数 MB）。バックエンドは Rust
3. **Python + PySide/Qt** — 開発速度が速い。パッケージング・配布がやや面倒
4. **C# + Avalonia** — .NET エコシステム。Windows 開発の王道。Avalonia でクロスプラットフォーム対応可能

## 決定

**Tauri v2** を採用する。

## 理由

- バイナリサイズが小さく、配布が容易
- GitHub Actions 向けに `tauri-action` が公式提供されており、CI/CD 構築が容易
- macOS / Windows / Linux のクロスプラットフォームビルドに初めから対応
- フロントエンドは Web 技術（React 等）で開発でき、UI の自由度が高い
- Rust バックエンドにより高いパフォーマンスと型安全性を得られる

## 影響・リスク

- Rust の学習コストが発生する
- Tauri v2 は比較的新しいフレームワークであり、情報量は Electron に比べて少ない

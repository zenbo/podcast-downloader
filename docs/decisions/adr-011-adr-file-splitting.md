# ADR-011: 設計判断記録（ADR）のファイル分割

| 項目 | 値 |
|------|-----|
| 日付 | 2026-02-22 |
| ステータス | 採用 |

## コンテキスト

設計判断記録を単一ファイル（`docs/06-decision-log.md`）で管理していたが、ADR の件数が増えるにつれてファイルが肥大化し、Claude Code のセッション開始時にコンテキストウィンドウを圧迫する問題が見込まれた。

## 検討した選択肢

1. **単一ファイルのまま** — 現状維持。全 ADR を1ファイルに記載し続ける
2. **ディレクトリ分割** — 一覧（README.md）と個別 ADR ファイルに分離する

## 決定

**選択肢 2: ディレクトリ分割** を採用する。

```
docs/decisions/
├── README.md                  # 一覧表 + 運用ルール
├── adr-001-app-framework.md   # 個別 ADR
├── ...
```

## 理由

- README.md（一覧）だけ読めば全体像を把握でき、コンテキストウィンドウを節約できる
- 必要な ADR のみ個別に参照すれば済む
- 一覧テーブルのタイトルに決定内容の要約を含めることで、詳細を読まなくても概要がわかる

## 影響・リスク

- CLAUDE.md および 02-architecture.md の参照先を更新済み
- 新規 ADR 追加時は個別ファイル作成 + README.md の一覧更新が必要（手順が1ステップ増える）

# ADR-005: 新着エピソードの判定ロジック

| 項目 | 値 |
|------|-----|
| 日付 | 2026-02-21 |
| ステータス | 採用 |

## コンテキスト

新着エピソードの判定基準を定義する。要件定義書の要件は「DL済み以後に配信されたエピソードを新着として表示する」。

## 検討した選択肢

1. **未ダウンロードエピソード全て** — ダウンロードしていないエピソードを全て新着とする
2. **最終 DL エピソードの配信日基準** — 最後にダウンロードしたエピソードの配信日以降に配信された未ダウンロードのエピソードを新着とする
3. **最終 DL 操作日時基準** — ダウンロード操作を行った日時以降に配信された未ダウンロードのエピソードを新着とする

## 決定

**選択肢 2: 最終 DL エピソードの配信日基準** を採用する。

## 判定ロジックの詳細

1. 当該番組のダウンロード履歴から、最後にダウンロードしたエピソードを特定する
2. そのエピソードの**配信日（published_at）**を基準日とする
3. 基準日以降に配信された**未ダウンロード**のエピソードを「新着」とする
4. **番組登録直後（一度もダウンロードしていない場合）**: 全エピソードを新着として扱う

## 理由

- 要件定義書の「DL済み以後に配信されたエピソード」の意図に最も合致する
- 過去のエピソードを遡ってダウンロードした場合に、それ以前のエピソードが新着として復活しない（配信日基準のため）

## 影響・リスク

- 古いエピソードを最初にダウンロードした場合、中間のエピソードが新着として表示されない可能性がある（ただし個別ダウンロードは常に可能）

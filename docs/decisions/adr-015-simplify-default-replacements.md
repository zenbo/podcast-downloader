# ADR-015: デフォルト文字置換ルールの簡素化

## ステータス

採用

## コンテキスト

現在のデフォルト `character_replacements` は Windows の禁止文字 7 種（`/` `:` `?` `"` `<` `>` `|`）すべてに個別ルールを設定している。しかし以下の問題がある。

1. `?` `"` `<` `>` `|` の `after` が空文字 `""` だが、`fallback_replacement` (`_`) との使い分けの根拠が設計ドキュメントに記載されていない
2. これらの文字は Podcast タイトルやエピソード名で意味のある区切り文字として使われることが少なく、空文字に置換すると単語が連結して可読性が下がるケースがある（例: `Q&A? Yes!` → `Q&A Yes!` vs `Q&A_ Yes!`）
3. 個別ルールの数が多いと設定画面の一覧が冗長になり、ユーザーが自分のルールを追加する際に分かりづらい

## 判断

`/` と `:` のみ個別ルールとして残し、それ以外の禁止文字は `fallback_replacement` (`_`) に統一する。

**変更後のデフォルト値:**

- `character_replacements`: `[{"before":"/","after":"-"},{"before":":","after":"-"}]`
- `fallback_replacement`: `"_"`（変更なし）

**個別ルールの選定基準:**

| 文字                 | 判断             | 理由                                                                                              |
| -------------------- | ---------------- | ------------------------------------------------------------------------------------------------- |
| `/`                  | 個別ルール (`-`) | 日付表記 `2024/01/15` → `2024-01-15` の可読性維持                                                 |
| `:`                  | 個別ルール (`-`) | ラベル区切り・時刻表記ともに `-` で許容範囲                                                       |
| `?` `"` `<` `>` `\|` | fallback に委任  | Podcast タイトルでの出現頻度が低く、個別に空文字置換するよりも `_` で統一した方が単語境界を保てる |

## 影響範囲

- `src-tauri/src/models/settings.rs` — `AppSettings::default()` の `character_replacements` を修正
- `docs/03-data-design.md` — デフォルト値テーブルの更新、after 選定基準の追記

**注意:** 既存ユーザーの設定ファイルには影響しない。`tauri-plugin-store` に保存済みの設定はそのまま使われ、デフォルト値は設定ファイルが存在しない場合（新規インストール時）にのみ適用される。

# ADR-002: フロントエンドフレームワークの選定

| 項目       | 値         |
| ---------- | ---------- |
| 日付       | 2026-02-21 |
| ステータス | 採用       |

## コンテキスト

Tauri v2 のフロントエンドとして使用する Web フレームワークを選定する。

## 検討した選択肢

1. **React + TypeScript** — エコシステム最大。情報量が豊富で問題解決しやすい
2. **Vue + TypeScript** — シンプルで書きやすい。Tauri 公式テンプレートあり
3. **Svelte + TypeScript** — 軽量で高速。ボイラープレート少ない

## 決定

**React + TypeScript** を採用する。ビルドツールには **Vite** を使用する。

## 理由

- エコシステムが最も大きく、ライブラリの選択肢が豊富
- TypeScript との親和性が高く、型安全な開発が可能
- 問題発生時に参照できる情報が多い
- Tauri 公式テンプレートが提供されている

## 影響・リスク

- Vue や Svelte に比べてボイラープレートが多い
- 状態管理ライブラリの選定が別途必要 → ADR-008 で決定済み
